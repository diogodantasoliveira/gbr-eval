### 1.6 Phase 2 — Grader bugs, golden-set-aware runner e integration (Claude)

Bugs encontrados e corrigidos durante integration smoke test:

| Bug | Fix |
|-----|-----|
| `_get_field` nao distinguia campo ausente de campo null | Sentinel `_MISSING` — `null == null` agora passa |
| `numeric_range` falhava em `capital_social: null` | Adicionado null-vs-null check |
| `_MISSING` sentinel vazava para JSON report | String "field missing" no details |
| Field names em extraction task YAMLs inconsistentes com golden sets | `numero_certidao`->`numero`, `data_validade`->`validade` |
| Citation grader misplaced em extraction task (matricula) | Removido — citation e responsabilidade do citation task |
| Field names em citation task YAMLs inconsistentes com golden sets | `citation.numero_certidao`->`citation.numero`, `citation.data_validade`->`citation.validade`, `citation.socios_percentual`->`citation.socios`, `citation.area`->`citation.area_total` |

Features adicionadas:
- **Golden-set-aware runner**: CLI flags `--golden-dir` e `--self-eval` no comando `run`
- `load_golden_cases()`: carrega cases de `golden/{skill}/case_0*.json`
- `_extract_document_type()`: extrai document_type do payload ou skill name (`certidao_trabalhista_v1` -> `certidao_trabalhista`)
- `run_task_against_golden_set()`: itera cases, merge citation data em self-eval, agrega scores
- `run_suite_with_golden()`: orquestra suite completa com golden sets
